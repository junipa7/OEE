설비 생산성 관리 시스템
데이터 보완 및 지표 산출 (Enrichment & Metrics) 상세 설계서

본 모듈은 설비에서 발생하는 단순한 원천(Raw) 메시지에 생산 관리 시스템(MOS/MES)의 비즈니스 컨텍스트(Lot, 제품코드 등)를 결합하고, 핵심 성능 지표인 CT(Cycle Time)와 TT(Tact Time)를 오차 없이 산출하는 역할을 수행합니다.

🔄 1. 상태 정보 보완 (Context Enrichment) 흐름 설계

단편적인 설비의 센서/제어기 데이터에 '어떤 제품을, 어떤 레시피로 만들고 있는지'에 대한 의미를 부여하는 과정입니다.

A. 컨텍스트(Context) 갱신 트리거

  • MOS/MES 이벤트 수신 시 (작업 지시, Track-In 등): CURR_LOT_ID, PROD_ID(기종 코드), PROC_CD, RECIPE_ID, TARGET_QTY 값을 즉시 업데이트합니다.
  • 설비 상태 알람 수신 시: EQP_STATUS (RUN, DOWN, IDLE) 값을 갱신합니다.

B. 데이터 병합(Join) 프로세스

  1. Raw 메시지 수신: 설비에서 메시지(예: 온도, 압력 등)가 들어오면 TB_RAW_MSG에 우선 적재합니다.
  2. 실시간 병합: 메시지가 들어온 즉시 백그라운드 워커(Worker) 프로세스가 해당 장비(EQP_ID)의 TB_CURRENT_CONTEXT 최신 레코드를 조회하여 Raw 데이터에 덧붙입니다.
  3. 최종 적재: 컨텍스트가 결합된 완전한 형태의 데이터가 TB_ENRICHED_LOG에 저장됩니다.

C. Race Condition (타이밍 불일치) 방어 로직

  • 문제 상황: 설비는 가동 시작 메시지를 보냈는데, 네트워크 지연으로 MOS의 Lot 정보가 아직 업데이트되지 않은 경우.
  • 해결 로직 (Wait & Retry): Raw 메시지 수신 시 CURRENT_CONTEXT의 Lot 정보가 비어있거나 최종 갱신 시간이 너무 오래되었다면, 해당 메시지를 메모리 큐에서 최대 3초간 대기(Wait)시킵니다. 3초 내에 MOS 이벤트가 도착하여 컨텍스트가 갱신되면 그때 병합을 수행합니다.

⏱️ 2. 핵심 지표 (CT / TT) 산출 알고리즘

설비의 실질적인 생산 속도를 측정하기 위해 Process Start와 Process End 이벤트를 기반으로 시간을 계산합니다.

  • CT (Cycle Time / 순수 가공 시간): 단일 제품/Wafer가 설비 내에서 실제로 가공된 시간.
    ➔ 계산식: 현재 Process End Time - 매핑된 Process Start Time
  • TT (Tact Time / 산출 간격 시간): 설비에서 제품이 배출되는 실제 간격 (물류 및 대기 시간 포함).
    ➔ 계산식: 현재 Process End Time - 직전(Previous) Process End Time

🛡️ 3. 3중 데이터 매핑 및 예외 처리 (방어 로직)

배치(Batch) 설비나 다중 챔버 설비의 경우, 여러 개의 Start 이벤트가 중첩되므로 End 이벤트와 정확히 짝을 맞추는 매핑(Mapping)과 추정(Estimation) 로직이 필수적입니다.

순위 매핑 방식 적용 조건 상세 처리 로직
1순위 고유 Key 기반 명시적 매핑
(Explicit Match)		 이벤트 페이로드에 식별자(MATERIAL_ID 등)가 포함된 경우 - Start 수신: TB_WIP_TRACKING에 [MATERIAL_ID, START_TIME] 저장
- End 수신: 동일한 MATERIAL_ID를 찾아 CT를 계산하고 해당 레코드를 삭제(Pop)
2순위 FIFO 매핑
(First-In-First-Out)		 Key는 없으나, 투입된 순서대로 배출되는 물리적 구조일 경우 - Start 수신: TB_WIP_TRACKING에 순서대로 누적
- End 수신: 대기열에서 가장 먼저 들어온(가장 오래된) START_TIME을 꺼내어 매핑
3순위 기준 시간 기반 역산 추정
(Fallback Estimation)		 네트워크 유실 등으로 인해 Start 이벤트 자체가 수신되지 않은 경우 - 산출식: 현재 Process End Time - 본사 표준 CT (또는 최근 평균 CT)
- 플래그 부여: TB_ENRICHED_LOG의 ESTIMATED_FLAG 컬럼에 'Y' 값을 부여하여 통계 분석 시 신뢰도 구분

🗑️ 가비지 컬렉션 (Garbage Collection / Memory Leak 방지)

상황: Start 이벤트만 수신되고 설비 에러나 수작업 배출로 인해 End 이벤트가 영원히 오지 않는 경우.

처리: TB_WIP_TRACKING을 주기적으로 스캔하여 대기 시간이 표준 CT의 3배를 초과한 항목은 가공 실패(Loss/Drop)로 간주하고 메모리에서 강제 삭제(Flush)한 뒤 에러 로그를 남깁니다.
