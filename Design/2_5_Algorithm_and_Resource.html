설비 생산성 관리 시스템
5. 알고리즘 파라미터 및 자원 최적화 상세 설계서

생산 현장(Edge)에서 방대한 양의 CT/TT 데이터를 본사(HQ)로 전송할 때, 원본 데이터를 모두 보내는 것은 불가능합니다. 본 문서는 데이터를 현장에서 압축(요약)하고 본사에서 수학적 오차 없이 병합(Merge)하기 위한 분산 통계 아키텍처 및 자원 할당 규격을 정의합니다.

📊 5.1. KLL Sketch 파라미터 설계 (분포 및 분위수 분석)

설비 데이터의 중앙값(Median), 95% 분위수(이상치 탐지용), 히스토그램을 추출하기 위해 KLL(Karnin, Lang, Liberty) Sketch 자료구조를 활용합니다. 성능과 용량의 균형을 위해 핵심 파라미터 K 값을 최적화합니다.

파라미터 / 특성 설정 값 및 상세 설명
최적화 K 값 K = 200 (기본 권장값 적용)
정확도 (Rank Error) 약 1.33% 보장 (예: 상위 5% 불량 CT를 찾을 때 4.9% ~ 5.1% 범위 내의 정밀도 도출)
메모리 및 전송 크기 30분간 데이터가 수만 건 누적되어도, 직렬화(Serialization)된 단일 KLL 객체의 크기는 최대 약 800 Bytes 이내로 제한됨.
병합(Merge) 특성 HQ 본사 DB에서 법인별/일자별 KLL 스케치 객체를 merge() 함수로 합쳐도 오차율(1.33%)이 누적되지 않고 동일하게 유지됨.

🧮 5.2. 분산 병합 알고리즘 (평균 및 분산 산출)

단순 합산 방식은 데이터가 커질수록 부동소수점 오류가 발생합니다. 원천 Raw 데이터 없이 요약된 값만으로 정확한 전체 분산과 표준편차를 구하기 위해 2단계 알고리즘을 적용합니다.

1단계: 현장 실시간 업데이트 Edge Server

스트리밍으로 들어오는 데이터를 Welford의 온라인 알고리즘을 통해 메모리에 축적합니다. 데이터 건마다 전체를 재계산하지 않고 상태 변수만 업데이트합니다.

  • 유지 상태 변수 (State Variables): n (데이터 개수), mean (현재 평균), M2 (평균과의 오차 제곱합)

2단계: 본사 데이터 병합 HQ Server

법인에서 전송받은 요약 데이터(A)와 기존 누적 데이터(B)를 통합할 때 Chan's Algorithm (1983)을 사용하여 수학적 오차 0%로 병합합니다.

[Chan's Merge Formula]

nAB = nA + nB
δ = meanB - meanA

meanAB = meanA + δ × ( nB / nAB )
M2AB = M2A + M2B + δ2 × ( nA × nB / nAB )

VarianceAB = M2AB / nAB

💻 5.3. Edge 서버 자원 관리 (Memory & CPU)

생산법인의 현장 서버(Edge)는 설비 제어 및 메시지 라우팅이 주 목적이므로, 분석 알고리즘이 시스템 리소스를 과도하게 점유하지 않도록 제한합니다.

  • 메모리 사용량 제한 (Max Heap Size):
    KLL 스케치 객체 및 Chan 상태 변수(n, mean, m2)를 유지하는 In-memory 캐시는 설비 당 최대 2MB를 초과하지 않도록 제한합니다.
  • 주기적 초기화 (Garbage Collection & Reset):
    매 30분마다 전송 스케줄러가 본사로 데이터를 보내고 HTTP 200 OK를 수신하면, 메모리에 적재되어 있던 해당 구간의 변수들을 즉시 0 또는 빈 객체로 초기화(Flush)하여 메모리 누수(Leak)를 원천 차단합니다.
