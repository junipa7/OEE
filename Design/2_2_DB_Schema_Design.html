설비 생산성 관리 시스템 DB 스키마 상세 설계서 (업데이트 반영)

🗄️ 1. 생산법인 (Edge / SQLite) 스키마 상세

현장의 고성능 서버(Linux)에 상주하며, 설비 메시지 수신, 상태 보완, CT/TT 실시간 산출 및 압축 큐잉을 담당하는 스키마입니다.

1.1. TB_CURRENT_CONTEXT (설비 실시간 상태 보완 테이블)

컬럼명 (Physical) 데이터 타입 PK 설명 (Logical & Role) 갱신 트리거 (Trigger)
EQP_ID VARCHAR Y 기준 장비(설비) 식별자 -
CURR_LOT_ID VARCHAR 현재 작업 중인 Lot ID MOS Track-In/Lot 변경 이벤트 수신 시
PROD_ID VARCHAR 현재 생산 기종 코드 (기존 MODEL_CD) MOS Track-In/Lot 변경 이벤트 수신 시
PROC_CD VARCHAR 현재 생산 공정 코드 MOS Track-In/Lot 변경 이벤트 수신 시
RECIPE_ID VARCHAR 현재 적용된 설비 레시피 ID 장비 Recipe Change 이벤트 수신 시
TARGET_QTY INTEGER 현재 Lot의 총 투입 목표 수량 MOS Track-In 이벤트 수신 시
DONE_QTY INTEGER 현재까지의 완성 수량 장비 Process End 이벤트 수신 시 (+1)
EQP_STATUS VARCHAR 설비 가동 상태 (RUN, IDLE, DOWN) 장비 Status Change 알람 수신 시
LAST_PRC_START_TIME DATETIME 가장 최근 Process Start 이벤트 발생 시간 장비 Process Start 수신 시. (CT 산출 기준점)
LAST_PRC_END_TIME DATETIME 가장 최근 Process End 이벤트 발생 시간 장비 Process End 수신 시. (TT 산출 기준점)
LAST_UPDATED DATETIME 레코드 최종 갱신 일시 위 컬럼들 중 하나라도 변경될 때마다 갱신

1.2. TB_WIP_TRACKING (재공 트래킹 임시 테이블)

컬럼명 (Physical) 데이터 타입 PK 설명 (Logical & Role)
MATERIAL_IDVARCHARY개별 제품, 웨이퍼, 글라스 고유 식별 Key
EQP_IDVARCHARY장비 식별자
START_TIMEDATETIMEProcess Start 발생 시간
INSERTED_ATDATETIME레코드 생성 시간

1.3. TB_ENRICHED_LOG (상태 보완 및 지표 산출 완료 로그 테이블)

컬럼명 (Physical) 데이터 타입 PK 설명 (Logical & Role)
LOG_IDVARCHARY보완된 로그의 고유 식별자
EQP_IDVARCHAR발생 설비 식별자
EVT_TYPEVARCHAR이벤트 유형
CURR_LOT_IDVARCHAR이벤트 발생 당시의 Lot 정보
PROD_IDVARCHAR이벤트 발생 당시의 기종 코드
RECIPE_IDVARCHAR이벤트 발생 당시의 레시피 정보
CT_VALFLOAT산출된 Cycle Time (초)
TT_VALFLOAT산출된 Tact Time (초)
ESTIMATED_FLAGCHAR(1)CT 추정 여부 (Y/N)
EVT_TIMEDATETIME이벤트 최종 완성 타임스탬프

1.4. TB_SEND_QUEUE (본사 전송용 30분 단위 대기열 테이블)

컬럼명 (Physical)데이터 타입PK설명 (Logical & Role)
SEQ_NOINTEGERY전송 큐 시퀀스
PAYLOAD_TYPEVARCHAR전송 데이터 유형
TARGET_URLVARCHAR본사 REST API 수신 엔드포인트
JSON_PAYLOADBLOBKLL 스케치 및 Chan 통계 페이로드
IS_SENTCHAR(1)본사 수신 확인 플래그
CREATED_ATDATETIME패키징 생성 일시

🏛️ 2. 본사 (HQ / Oracle) 스키마 상세

요청에 따라 사업부코드(HEADQUARTERCODE) 및 제품코드(GODS_CODE)를 추가하였으며, 명칭을 새롭게 반영했습니다.

2.1. TB_HQ_EQP_MASTER (글로벌 설비 마스터 테이블)

컬럼명 (Physical) 데이터 타입 PK 설명 (Logical & Role)
HEADQUARTERCODE VARCHAR2 Y 사업부코드 (신규 추가)
SITE_CODE VARCHAR2 Y 법인코드 (기존 CORP_ID)
EQP_ID VARCHAR2 Y 글로벌 장비 식별자
LINE_CD VARCHAR2 라인 코드
EQP_GROUP VARCHAR2 장비 그룹 (동일 스펙/유형 분석용)
USE_YN CHAR(1) 장비 사용 여부 (Y/N)

2.2. TB_HQ_STD_INFO (표준 시간 및 기준정보 관리 테이블)

컬럼명 (Physical) 데이터 타입 PK 설명 (Logical & Role)
HEADQUARTERCODE VARCHAR2 Y 사업부코드 (신규 추가)
SITE_CODE VARCHAR2 Y 법인코드
GODS_CODE VARCHAR2 Y 제품코드 (신규 추가)
PROD_ID VARCHAR2 Y 기종 코드 (기존 MODEL_CD)
PROC_CD VARCHAR2 Y 공정 코드
RECIPE_ID VARCHAR2 Y 레시피 ID
STD_CT NUMBER 글로벌 표준 Cycle Time (Standard Time)
UPDATED_DT DATE 기준 시간 최종 변경 일시

2.3. TB_AGGR_RSLT (통합 실적 및 OEE 집계 테이블)

컬럼명 (Physical) 데이터 타입 PK 설명 (Logical & Role)
HEADQUARTERCODE VARCHAR2 Y 사업부코드 (신규 추가)
SITE_CODE VARCHAR2 Y 법인코드
EQP_ID VARCHAR2 Y 장비 식별자
GODS_CODE VARCHAR2 제품코드 (해당 실적의 주요 제품군)
PROD_ID VARCHAR2 기종 코드
WORK_DATE VARCHAR2 Y 작업 일자 (YYYYMMDD 형식)
TIME_WINDOW VARCHAR2 Y 30분 단위 구분 (예: 1200_1230)
TOTAL_RUN_TIME NUMBER 해당 30분 내 총 가동 시간 (초)
TOTAL_STOP_TIME NUMBER 해당 30분 내 총 비가동(알람/에러) 시간 (초)
TOTAL_IDLE_TIME NUMBER 해당 30분 내 총 대기 시간 (초)
DONE_QTY NUMBER 해당 30분 내 달성한 생산 완료 수량
OEE_SCORE NUMBER 계산된 설비종합효율(%)

2.4. TB_STAT_METRICS (통계 지표 KLL/Chan 원천 저장소)

컬럼명 (Physical) 데이터 타입 PK 설명 (Logical & Role)
HEADQUARTERCODE VARCHAR2 Y 사업부코드 (신규 추가)
SITE_CODE VARCHAR2 Y 법인코드
EQP_ID VARCHAR2 Y 장비 식별자
GODS_CODE VARCHAR2 Y 제품코드
PROD_ID VARCHAR2 Y 기종 코드
RECIPE_ID VARCHAR2 Y 레시피 식별자
TIME_WINDOW VARCHAR2 Y 30분 단위 집계 시간
METRIC_TYPE VARCHAR2 Y 지표 유형 ('CT' 또는 'TT')
CHAN_COUNT NUMBER 해당 30분간 집계된 데이터 개수 (n)
CHAN_MEAN NUMBER 해당 30분간의 평균 시간 (Mean)
CHAN_M2 NUMBER 해당 30분간의 오차 제곱합 (M2) - 분산 도출용
KLL_SKETCH_BLOB BLOB 직렬화(Serialized)된 바이너리 객체 데이터 (중앙값 및 분포 도출용)
INSERTED_DT DATE 본사 데이터베이스 적재 시간
