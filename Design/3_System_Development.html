📄 시스템 개발 (System Development) 효율적인 개발을 위해 전체 시스템을 4가지 핵심 모듈로 나누어 진행 순서를 제안해 드립니다. 1. 생산법인(Edge) 실시간 처리 및 상태 관리 모듈 개발 - 설비 메시지 파싱 및 지표 산출: Linux 기반의 고사양 서버 환경에서 설비 메시지를 파싱하고 실시간으로 CT(Cycle Time)와 TT(Tact Time) 및 일별 생산실적을 집계하는 모듈을 구현합니다. - 실시간 상태(Context) 매핑: MOS와 연동하여 현재 작업 중인 Lot ID, 투입/완성 수량 등의 정보를 실시간으로 SQLite에 반영하고 설비 메시지와 결합하는 기능을 개발합니다. 2. 생산법인(Edge) 데이터 압축 및 OEE 추정 모듈 개발 - 알고리즘 연산 적용: 인메모리 환경에서 KLL 스케치 및 Chan 알고리즘을 적용한 데이터 압축 및 실시간 OEE 추정 모듈을 개발합니다. - 메모리 최적화 로직: 앞서 설계한 K=200 파라미터를 적용하고, Welford의 온라인 알고리즘을 활용해 상태값(n, mean, M2)만 업데이트하여 메모리 사용량을 최소화하는 코드를 구현합니다. 3. 네트워크 전송 및 장애 대응 모듈 개발 (Edge ↔ HQ) - 배치 전송 스케줄러: 30분 주기의 배치 전송 모듈을 개발합니다. KLL 스케치 객체를 Base64로 직렬화하고 최종 JSON 페이로드에 Zstandard(Zstd) 압축을 적용하는 로직을 작성합니다. - Store & Forward 구현: 네트워크 단절 시 데이터 유실을 방지하는 전송 대기 큐(Queue)를 개발합니다. 통신 복구 시 누적된 패키지를 순차적으로 일괄 전송(Retry)하는 방어 로직을 포함합니다. 4. 본사(HQ) 통합 서버 모듈 개발 - 기준정보 배포 파이프라인: 글로벌 기준정보를 관리하고 표준 코드(기종, 공정, ST)를 7개 생산법인으로 안전하게 배포(Sync)하는 기능을 구현합니다. - 통계 병합 및 ST 산출: 7개 법인에서 수집된 요약 데이터를 바탕으로 Standard Time(표준 시간)을 산출하는 통계 처리 로직을 개발합니다. Chan의 병합 알고리즘 공식을 코드화하여 오차 없이 평균과 분산을 역산하는 것이 핵심입니다. 🛠️ 추천 기술 스택 (예시) 시스템 영역 추천 언어/프레임워크 선정 사유 Edge (생산법인) C++ 또는 Python 인텔® 코어™ Ultra의 멀티코어 연산을 활용한 KLL/Chan 수학적 알고리즘 고속 처리 Edge DB SQLite 로컬 환경에서의 파일 기반 초고속 I/O 처리 및 인메모리 관리 HQ (본사) Java (Spring Boot) 안정적인 글로벌 REST API 서버 구축 및 이기종 DB 연동의 엔터프라이즈 범용성 HQ DB Oracle 대규모 글로벌 통합 시계열 통계 데이터의 안전한 트랜잭션 관리
