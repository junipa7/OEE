4. 본사(HQ) 통합 서버 모듈 개발 (Spring Boot)

1. 모듈 개요 (Module Overview)

본사 통합 서버는 7개 글로벌 생산법인에서 Zstd로 압축하여 전송한 통계 페이로드를 수신하고, 분산된 통계 데이터를 로우 데이터(Raw Data) 없이 수학적으로 정확하게 병합하는 역할을 수행합니다.

  • 압축 해제 및 복원: Zstd 페이로드를 압축 해제하고, Base64로 인코딩된 KLL 스케치 객체를 Java 환경에서 복원합니다.
  • 통계 병합 (Chan's Algorithm): 각 법인의 평균(Mean)과 분산(Variance)을 병합하여 글로벌 통합 표준 시간(Standard Time, ST)을 산출합니다.
  • 기준정보 배포 (Master Data Sync): 본사에서 관리하는 설비 표준 코드 및 목표 ST를 각 법인으로 일괄 배포하는 파이프라인을 운영합니다.

2. 핵심 구현 1: Zstd 페이로드 수신 및 압축 해제

Spring Boot의 REST Controller에서 application/zstd 형태의 바이너리 스트림을 받아 해제하는 로직입니다. zstd-jni 라이브러리를 활용합니다.

import com.github.luben.zstd.ZstdInputStream;
import com.fasterxml.jackson.databind.ObjectMapper;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/v1/telemetry")
public class EdgeDataReceiverController {

    private final ObjectMapper objectMapper = new ObjectMapper();

    @PostMapping(value = "/upload", consumes = "application/zstd")
    public ResponseEntity<String> receiveData(InputStream requestStream) {
        try (ZstdInputStream zstdStream = new ZstdInputStream(requestStream)) {
            // 1. Zstd 압축 해제 및 JSON 파싱
            EdgePayloadDTO payload = objectMapper.readValue(zstdStream, EdgePayloadDTO.class);
            
            // 2. Service 레이어로 전달하여 병합 로직 수행
            statisticsService.processPayload(payload);
            
            return ResponseEntity.ok("SUCCESS");
        } catch (Exception e) {
            return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body("FAIL");
        }
    }
}

3. 핵심 구현 2: 분산 통계 병합 알고리즘 (Chan's Update Formula)

두 개 이상의 법인(또는 시간대)에서 집계된 (Count, Mean, Variance) 데이터 셋을 원본 데이터 없이 정확하게 하나로 합치는 로직입니다. 분산 병합을 위해 Chan의 병합 알고리즘을 적용합니다.

public class StatisticalMerger {

    public static MergedStat merge(Stat stat1, Stat stat2) {
        long n1 = stat1.getCount();
        long n2 = stat2.getCount();
        long n = n1 + n2;

        if (n == 0) return new MergedStat(0, 0.0, 0.0);

        // 평균 병합 (Combined Mean)
        double delta = stat2.getMean() - stat1.getMean();
        double combinedMean = stat1.getMean() + (delta * n2 / n);

        // M2 (Sum of squares of differences) 복원 및 병합
        // Variance = M2 / (n - 1) 이므로 역산하여 M2를 구함
        double m2_1 = stat1.getVariance() * (n1 - 1);
        double m2_2 = stat2.getVariance() * (n2 - 1);
        
        // Chan's Algorithm을 활용한 M2 병합
        double combinedM2 = m2_1 + m2_2 + (delta * delta * n1 * n2 / n);
        
        // 통합 분산(Combined Variance) 계산
        double combinedVariance = combinedM2 / (n - 1);

        return new MergedStat(n, combinedMean, combinedVariance);
    }
}

※ KLL 스케치 병합의 경우 Apache DataSketches Java 라이브러리의 KllFloatsSketch.merge(otherSketch) 메서드를 호출하여 간단히 글로벌 P50(중앙값) 및 P99 분포를 산출할 수 있습니다.

4. 핵심 구현 3: 기준정보 (Master Data) 배포 파이프라인

본사에서 확립된 표준 CT(Cycle Time)와 기종 정보를 7개 법인으로 동기화하기 위한 엔드포인트입니다. Edge 노드가 주기적으로 이 API를 호출하여 SQLite 캐시를 업데이트합니다.

@RestController
@RequestMapping("/api/v1/master-data")
public class MasterDataSyncController {

    @GetMapping("/sync/{corpCode}")
    public ResponseEntity<MasterDataResponse> getUpdatedMasterData(
            @PathVariable String corpCode,
            @RequestParam String lastSyncTimestamp) {
        
        // 본사 Oracle DB에서 lastSyncTimestamp 이후 변경된 기준정보만 추출 (Delta Sync)
        List<StandardTimeDTO> updatedST = masterDataService.getUpdatedST(corpCode, lastSyncTimestamp);
        
        MasterDataResponse response = new MasterDataResponse(updatedST);
        return ResponseEntity.ok(response);
    }
}

5. 본사 통합 서버 개발 시 체크포인트

  • 글로벌 Timezone 일원화: 7개 국가의 법인에서 올라오는 데이터의 타임스탬프는 모두 UTC (Coordinated Universal Time)로 통일하여 Oracle DB에 적재해야 시계열 통계 분석 시 오류가 발생하지 않습니다.
  • Oracle DB 파티셔닝 (Partitioning): 설비별/법인별 요약 데이터라도 장기적으로 쌓이면 테이블이 거대해집니다. EVENT_DATE를 기준으로 월 단위 범위 파티셔닝(Range Partitioning)을 적용하여 조회 성능을 확보해야 합니다.
  • 무결성 보장: Edge에서 보낸 JSON 페이로드의 해시값(SHA-256 등)을 헤더에 포함시켜, 네트워크 전송 중 데이터가 위변조되거나 깨지지 않았는지 검증하는 체크섬(Checksum) 로직 추가를 권장합니다.
