생산법인(Edge) 실시간 처리 및 상태 관리 모듈 개발

1. 전체 데이터 흐름 (Data Flow)

  1. Event 수신: 설비로부터 생산 시작(Start), 종료(End), 알람(Alarm) 등의 원천 로그 수신 (TCP/MQTT 등).
  2. Context 매핑: 메모리 상의 MOS 데이터(현재 Lot ID, 투입 수량 등)와 설비 이벤트를 결합.
  3. 지표 산출: 시간차 연산을 통해 실시간 CT(Cycle Time) 및 TT(Tact Time) 계산.
  4. 로컬 저장: 처리된 결과물을 SQLite에 고속 저장.

2. 핵심 구현 1: 설비 메시지 파싱 및 지표(CT/TT) 산출 로직

설비에서 들어오는 데이터를 파싱하고, 이전 작업의 타임스탬프와 비교하여 지표를 산출하는 Python 비동기 클래스 구조입니다.

import asyncio
from datetime import datetime

class EquipmentMonitor:
    def __init__(self):
        # 설비별 마지막 작업 완료 시간을 메모리에서 관리 (고속 연산 용도)
        self.last_end_time = {} 

    async def process_message(self, raw_msg):
        """설비 수신 메시지 파싱 및 지표 산출"""
        # 1. 메시지 파싱 (가정: JSON 형태)
        data = self.parse_json(raw_msg)
        eqp_id = data['equipment_id']
        event_time = datetime.fromisoformat(data['timestamp'])
        event_type = data['event_type'] # 'START' or 'END'

        ct, tt = None, None

        # 2. 지표 산출 로직
        if event_type == 'END':
            # CT 산출 (현재 END 시간 - 현재 START 시간)
            start_time = datetime.fromisoformat(data['start_time'])
            ct = (event_time - start_time).total_seconds()

            # TT 산출 (현재 END 시간 - 이전 END 시간)
            if eqp_id in self.last_end_time:
                tt = (event_time - self.last_end_time[eqp_id]).total_seconds()
            
            # 상태 업데이트
            self.last_end_time[eqp_id] = event_time

        return {
            "eqp_id": eqp_id,
            "timestamp": event_time,
            "ct": ct,
            "tt": tt
        }

3. 핵심 구현 2: 실시간 Context 매핑 및 SQLite 고속 처리

MOS(생산관리시스템)의 작업 지시 정보를 실시간 설비 데이터와 결합합니다. SQLite의 I/O 병목을 해결하기 위해 WAL(Write-Ahead Logging) 모드를 사용합니다.

💡 SQLite 테이블 스키마 설계 (예시)

CREATE TABLE FACT_PRODUCTION (
    SEQ INTEGER PRIMARY KEY AUTOINCREMENT,
    EQP_ID TEXT NOT NULL,
    LOT_ID TEXT NOT NULL,         -- MOS Context 매핑
    MODEL_CODE TEXT,              -- MOS Context 매핑
    EVENT_TIME DATETIME NOT NULL,
    CYCLE_TIME REAL,
    TACT_TIME REAL,
    PROD_QTY INTEGER              -- 실적 수량
);

💡 Python 기반 SQLite 고속 저장 및 Context 매핑 로직

import sqlite3

class ContextManagerDB:
    def __init__(self, db_path="edge_local.db"):
        self.conn = sqlite3.connect(db_path, isolation_level=None)
        # SQLite 성능 최적화: WAL 모드 활성화로 동시성 확보
        self.conn.execute("PRAGMA journal_mode=WAL;")
        self.conn.execute("PRAGMA synchronous=NORMAL;")
        
        # 최신 MOS 상태값을 메모리에 캐싱
        self.mos_context_cache = {}

    def update_mos_context(self, eqp_id, lot_id, model_code):
        """MOS 연동 데몬이 주기적으로 호출하여 최신 Context 업데이트"""
        self.mos_context_cache[eqp_id] = {
            "lot_id": lot_id,
            "model_code": model_code
        }

    def insert_production_record(self, metrics_data):
        """산출된 지표와 Context를 결합하여 DB 저장"""
        eqp_id = metrics_data['eqp_id']
        context = self.mos_context_cache.get(eqp_id, {"lot_id": "UNKNOWN", "model_code": "UNKNOWN"})

        query = """
            INSERT INTO FACT_PRODUCTION (EQP_ID, LOT_ID, MODEL_CODE, EVENT_TIME, CYCLE_TIME, TACT_TIME)
            VALUES (?, ?, ?, ?, ?, ?)
        """
        self.conn.execute(query, (
            eqp_id, 
            context['lot_id'], 
            context['model_code'], 
            metrics_data['timestamp'], 
            metrics_data['ct'], 
            metrics_data['tt']
        ))

4. Edge 모듈 개발 시 체크포인트

  • 메모리 누수(Memory Leak) 방지: last_end_time이나 캐시 딕셔너리가 무한정 커지지 않도록 주기적으로 사용하지 않는 설비 ID를 정리하는 Garbage Collection 로직 필요.
  • 예외 처리(Exception Handling): 타임스탬프 역전이나 비정상적인 CT/TT 값(음수 또는 기준치 10배 이상)은 Drop 하거나 별도 Error Log로 분리하는 필터링 로직 추가.
