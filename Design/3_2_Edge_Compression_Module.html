2. 생산법인(Edge) 데이터 압축 및 OEE 추정 모듈 개발

1. 모듈 개요 (Module Overview)

본 모듈은 Edge 환경에서 수집되는 초당 수백 건의 Cycle Time(CT) 및 Tact Time(TT) 데이터를 본사로 전부 전송하지 않고, 인메모리(In-Memory) 상태에서 실시간으로 압축 및 통계 처리를 수행하는 핵심 엔진입니다.

  • 메모리 최적화: 전체 데이터를 배열에 저장하지 않고, 상태값만 업데이트하는 스트리밍 알고리즘 적용.
  • 네트워크 비용 절감: KLL 스케치 알고리즘을 통해 수백만 건의 데이터를 수 KB 수준의 바이트 배열로 압축.
  • OEE 추정: 가동률(A), 성능효율(P), 양품률(Q)을 실시간 집계하여 설비 종합 효율 산출.

2. 핵심 구현 1: Welford의 온라인 알고리즘 (평균/분산 스트리밍 연산)

메모리 부족(OOM) 방지를 위해, 과거 데이터를 저장하지 않고 새로 들어오는 데이터만으로 평균(Mean)과 분산(Variance)을 정확하게 계산하는 Welford's Online Algorithm을 구현합니다.

class WelfordAggregator:
    def __init__(self):
        # 상태값만 메모리에 유지 (전체 데이터 배열 저장 안 함)
        self.count = 0
        self.mean = 0.0
        self.M2 = 0.0 # Sum of squares of differences from the current mean

    def update(self, value):
        """새로운 CT/TT 데이터가 들어올 때마다 상태값 업데이트"""
        self.count += 1
        delta = value - self.mean
        self.mean += delta / self.count
        delta2 = value - self.mean
        self.M2 += delta * delta2

    def get_mean(self):
        return self.mean

    def get_variance(self):
        """표본 분산(Sample Variance) 반환"""
        if self.count < 2:
            return 0.0
        return self.M2 / (self.count - 1)

3. 핵심 구현 2: Apache DataSketches (KLL 스케치) 적용

데이터의 분포(Distribution)와 중앙값(P50), 99분위수(P99) 등을 추정하기 위해 KLL(Karnin, Lang, Liberty) 스케치를 도입합니다. 파라미터 K=200을 설정하여 정확도(오차율 ~1.33%)와 메모리 사용량(수 KB)의 최적 균형을 맞춥니다.

# pip install datasketches 라이브러리 활용
from datasketches import kll_floats_sketch

class KllCompressor:
    def __init__(self, k=200):
        # K=200 설정으로 Edge 서버의 메모리 부하 최소화
        self.sketch = kll_floats_sketch(k)

    def add_data(self, value):
        """실시간 지표 스트리밍 입력"""
        if value is not None:
            self.sketch.update(value)

    def get_quantiles(self):
        """P50(중앙값), P90, P99 추출"""
        if self.sketch.is_empty():
            return None
        return {
            "P50": self.sketch.get_quantile(0.5),
            "P90": self.sketch.get_quantile(0.9),
            "P99": self.sketch.get_quantile(0.99)
        }

    def get_serialized_data(self):
        """본사 전송을 위해 직렬화 (이후 Zstd로 추가 압축 진행)"""
        return self.sketch.serialize()

4. OEE (설비종합효율) 실시간 추정 로직 결합

위에서 산출된 통계 데이터와 앞선 모듈(SQLite)의 Context 데이터를 결합하여 Edge 단에서 즉각적인 OEE 모니터링이 가능하도록 구성합니다.

  • Availability (가동률): (계획 정지 시간을 제외한 실제 가동 시간) / (부하 시간)
  • Performance (성능효율): (이론적 CT * 총 생산량) / (실제 가동 시간) -> Welford 연산의 Mean 값을 활용
  • Quality (양품률): (양품 수량) / (총 생산 수량)

※ Edge에서는 일정 주기(예: 30분, 1시간)마다 위의 WelfordAggregator와 KllCompressor 객체를 새로 생성하여 메모리를 비우고(Reset), 직렬화된 요약본만 본사(HQ)로 전송하는 스케줄러와 연동됩니다.
